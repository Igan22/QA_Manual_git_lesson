1. git clone — Клонування репозиторію
Ця команда використовується один раз на самому початку роботи, щоб завантажити (скопіювати) готовий проєкт із віддаленого сервера (наприклад, з GitHub) на твій локальний комп'ютер.

Що вона робить: Створює на комп'ютері папку з назвою проєкту, переносить туди всі файли та всю історію змін (комітів).

Приклад: git clone https://github.com/user/repository.git

2. git pull — Оновлення локальної версії
Цю команду варто виконувати щоразу перед початком роботи або написанням нового коду.

Що вона робить: Звертається до віддаленого сервера, перевіряє, чи не зробив хтось із колег (або викладач) нових змін у проєкті, завантажує їх та автоматично об'єднує з твоїми локальними файлами. Це допомагає уникнути конфліктів у коді.

3. git status — Перевірка стану
Головний інформатор тестувальника та розробника. Її можна викликати в будь-який момент.

Що вона робить: Показує, які файли ти змінив, які створив нові (untracked), а які видалив. Вона підкаже, чи готовий проєкт до збереження, чи потрібно спочатку додати файли до індексу.

4. git add . — Підготовка файлів (Індексація)
Що вона робить: Крапка . наприкінці означає "додати всі змінені та нові файли з поточної папки". Команда переносить ці файли до підготовчої зони (Staging Area). Git тепер знає, що саме ці файли ти плануєш зберегти у наступному кроці.

5. git commit -m "Add homework file" — Фіксація змін
Що вона робить: Робить "знімок" (snapshot) усіх підготовлених файлів і назавжди зберігає їх у базу даних твого локального репозиторію (на твоєму комп'ютері).

Прапорець -m (message) дозволяє в лапках написати короткий коментар про те, що саме було зроблено (наприклад: «Додав файл із домашнім завданням»).

6. git push — Відправка на сервер
Фінальний крок роботи.

Що вона робить: "Виштовхує" всі твої локальні коміти (збереження) на віддалений сервер (GitHub). Тільки після цієї команди твій викладач або колеги зможуть побачити твою виконану роботу на сайті GitHub.
