git init — створює новий локальний репозиторій Git у поточній папці.
git clone <URL> — копіює існуючий віддалений репозиторій на ваш комп'ютер.
git status — показує стан файлів: які змінені, а які ще не додані до відстеження.
git add <файл> або git add . — додає зміни до індексу (готує їх до збереження).
git commit -m "Коментар" — фіксує підготовлені зміни та створює їх «знімок» (коміт) у вашій історії.
git push — відправляє ваші зафіксовані (зроблені через commit) локальні зміни на віддалений сервер.
git pull — завантажує останні оновлення з віддаленого сервера та об'єднує їх з вашим поточним кодом.
git fetch — перевіряє наявність нових змін на віддаленому сервері, не завантажуючи самі файли.
git branch — виводить список локальних гілок у проекті.
git branch <назва_гілки> — створює нову гілку.
git checkout <назва_гілки> — перемикає вас на вказану гілку.
git merge <назва_гілки> — об'єднує зміни з вибраної гілки в поточну.
